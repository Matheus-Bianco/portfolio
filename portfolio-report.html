Projetos

01 — Plataforma de Indicadores de Risco Educacional

A SED Sergipe necessitava de uma abordagem sistemática para identificar escolas e estudantes em risco em múltiplas dimensões (abandono, reprovação, frequência, desempenho). O desafio era integrar dados de múltiplas fontes: sistema estadual, SAEB, SAESE, Plurall e INSE.

Plataforma integrada em Python que processa ~597 MB de registros e gera indicadores compostos por escola. Pipeline de ETL com extração, limpeza, cálculo de indicadores (abandono, reprovação, frequência, notas, distorção idade-série), classificação de risco e cruzamento com SAEB/SAESE/Plurall. Produto final: planilha consolidada com dicionário de dados.

Python · Pandas · ETL · SAEB · Looker Studio

02 — Painel de Monitoramento de Estudantes Imigrantes

Joinville atende 3.923 alunos imigrantes de 47 nacionalidades em 170 escolas. A administração necessitava de um instrumento para mapear a população, compreender distribuição geográfica e escolar, acompanhar desempenho e subsidiar políticas de acolhimento.

Sistema em duas camadas: pipeline Python processando 6 anos de registros (2021–2026), identificação de imigrantes por nacionalidade, deduplicação e geração de 18+ CSVs analíticos. Aplicação web com mapa Leaflet.js (170 escolas), 10+ gráficos interativos (nacionalidade, gênero, curso, turno, etapa), integração com programa Innovamat e geração de relatórios PDF por escola.

Python · Google Apps Script · Leaflet.js · Chart.js

03 — Sistema de Acompanhamento de Dados GA/SE

A Gerência de Avaliação da SED Sergipe necessitava de um sistema integrado para acompanhar movimentação (abandonos, transferências, evasões), reprovação por componente curricular e frequência escolar. Dados na API da SED, mas exigiam consolidação e painel web com login.

Painel web com extração via API, tratamento em Python e três módulos: Movimentação (abandonos, transferências, evasões, cancelamentos com histórico temporal), Reprovação (rendimento, reprovação por componente/área, ranking de escolas), Frequência (monitoramento de infrequência por faixa). Interface segura com autenticação e filtros por ano, DRE e escola.

Python · API · Google Apps Script

04 — Sistema de Gestão de Avaliações Diagnósticas

A rede estadual de Sergipe implementou avaliações diagnósticas com a plataforma Plurall em 201 escolas e 10 DREs. O ciclo — agendamento, aplicação, consolidação — exigia um sistema coordenado para gerenciar logística e dar visibilidade em tempo real.

Aplicação web em Google Apps Script com login (Admin, DRE, Escola), módulo de agendamento (datas e turnos para 1ª–3ª série em LP e Matemática), processamento de resultados (upload Plurall/Somos, mapeamento 221 escolas), dashboard de agendamento e dashboard de resultados (participação, desempenho por escola/série/disciplina). Banco em Google Sheets com 7 abas especializadas.

Google Apps Script · Bootstrap · Chart.js · Google Sheets

05 — Painel de Recomposição de Aprendizagens

Joinville implementou programa de recomposição para estudantes com defasagens. A gestão demandava um painel para acompanhar participação, medir progresso nas atividades de recuperação e gerar relatórios acionáveis para gestores escolares.

Dashboard web com backend Google Apps Script (DataService.js, Code.js), interface responsiva com login, sidebar, KPIs, busca e filtragem de alunos. Pré-processamento Node.js para CSVs via Google APIs. Integração jsPDF para geração automatizada de relatórios com tabelas formatadas por escola. Acompanhamento de progresso e atividades.

Google Apps Script · Node.js · jsPDF · Google APIs

06 — Sistema de Monitoramento Temporal de Risco

A SED Sergipe gerava relatórios Excel diários de risco por aluno, escola, série e turma. A análise isolada não permitia identificar tendências nem avaliar se as intervenções reduziam o risco ao longo do tempo.

Sistema automatizado que transforma relatórios diários em ferramenta longitudinal: extração de datas dos nomes de arquivos (DD.MM.AAAA, DD_MM), processamento padronizado (risco_reprovacao = "Sim"), agregação por escola/INEP, série, turma e data. Saídas: base_temporal_detalhada.xlsx e resumo_temporal.xlsx com totais diários de alunos em risco, escolas afetadas e turmas impactadas.

Python · Pandas · Séries Temporais · Excel

07 — Sistema de Priorização de Escolas para o SAEB

A preparação para o SAEB 2025 exigia identificar e priorizar escolas que mais necessitavam de apoio. Com centenas de escolas, uma seleção subjetiva não alocaria eficientemente os recursos limitados.

Ferramenta de priorização multicritério: definição de critérios baseados em indicadores de desempenho e risco, análise em Python com pontuações compostas, dashboard HTML interativo e aplicação web em Google Apps Script com sidebar para a equipe gestora. Cenários de priorização (8 ou 10 escolas/articulador), filtros por DRE e ajuste de pesos (Participação, Rendimento).

Python · Google Apps Script · HTML · Análise Multicritério

08 — Painel de Indicadores Educacionais — IDEB & Censo

A SED Sergipe necessitava de infraestrutura que consolidasse IDEB e Censo Escolar em painel analítico para análises comparativas entre estados, regiões e séries históricas. Dados de múltiplas fontes, anos e formatos.

Repositório integrado com séries históricas do IDEB (2022–2024) e microdados do Censo 2023. Pipeline de padronização (códigos UF, etapas, definições), cruzamento entre IDEB, infraestrutura do Censo e Indicadores Compostos de Gestão (ICG). Documentação técnica completa do pipeline e definições de indicadores.

Python · Dados INEP · CSV · Pipeline de Dados
